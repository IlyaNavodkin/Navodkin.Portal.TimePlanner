# Реализация графика таймлайнов

## 1. Что нужно от графика
- Горизонтальная временная ось по дням/неделям.
- Вертикальные строки по чарджам (с группировкой по проектам).
- Прямоугольники-таймлайны с сотрудниками.
- Прокрутка по горизонтали.
- Клик по строке для добавления назначения.
- Потенциально drag/resize блоков в будущем.

## 2. Варианты реализации

### Вариант A: CSS Grid + absolute blocks (рекомендуется)
Подход:
- рисуем сетку календаря обычным DOM (`div`) через CSS Grid;
- блоки таймлайна позиционируем абсолютом в координатах `left/width`;
- горизонтальный скролл у контейнера.

Плюсы:
- быстро реализовать в Nuxt + Tailwind;
- проще интеграция с popover, тултипами, событиями мыши;
- хорошая читаемость и поддержка.

Минусы:
- при очень больших объемах нужна виртуализация строк.

Когда выбирать:
- текущий MVP/production v1 для планировщика.

### Вариант B: SVG
Подход:
- сетка и блоки в `svg` (rect/text).

Плюсы:
- точный контроль рендера;
- удобные масштабирование и геометрия.

Минусы:
- сложнее UI-обвязка (popover/focus/keyboard);
- сложнее стилизация Tailwind-подходом.

### Вариант C: Canvas
Подход:
- все отрисовывается на `canvas`.

Плюсы:
- высокая производительность при большом числе объектов.

Минусы:
- сложная интерактивность и accessibility;
- больше кода для hit-testing и редактирования.

### Вариант D: готовая библиотека Gantt/Timeline
Примеры: `vis-timeline`, `dhtmlx-gantt`, `frappe-gantt`.

Плюсы:
- быстро получить много функций.

Минусы:
- ограничения кастомизации под ваш UX;
- лицензионные/весовые риски;
- сложнее встраивание в специфический layout.

## 3. Рекомендация
Для вашего кейса выбрать `Вариант A`:
- Nuxt + Tailwind нативно подходят;
- по макету нужны прямоугольники и поповеры, это проще всего в DOM;
- можно добавить виртуализацию строк (например, `@tanstack/virtual`) при росте объема.

## 4. Технический набросок координат
1. Выбираем `timelineStart` (сегодня) и `dayWidth` (например, 40 px).
2. Для блока (данные приходят как массив `days[]`):
   - `firstDay = min(days[])`;
   - `lastDay = max(days[])`;
   - `offsetDays = firstDay - timelineStart`;
   - `durationDays = lastDay - firstDay + 1`;
   - `left = offsetDays * dayWidth`;
   - `width = durationDays * dayWidth`.
3. Строка чарджа хранит `rowIndex`, верхняя координата:
   - `top = rowIndex * rowHeight`.

## 5. Popover выбора сотрудника
- Компонент с:
  - input поиска;
  - виртуализируемым списком сотрудников;
  - кнопкой подтверждения создания assignment.
- Источник сотрудников: `/api/managers/:id/employees` (проксировано через backend).
